App Flows

Single Prospect Flow
1
Entry — Company lookup or manual Type company → Exa finds decision-makers. Or enter name, email, LinkedIn manually. Exa
2
People Picker Select who to target from Exa-returned LinkedIn profiles grid. UI only
3
Signal Picker Exa fetches company news + person signals. Pick the best hook or skip. Exa Redis dup check
4
Generate Groq writes email body, 3 subject lines, landing page copy, and quality scores in one call. Groq LLaMA 3.3 70B
5
Review & Edit Edit email + LP content inline. Pick subject line. Scores flagged if < 6/10. UI only
6
Push Apollo gets the contact + sequence enrol. Slack fires a notification. History saved to Redis. Apollo Slack Redis
Bulk Generate Flow
1
Find Companies ICP description → AI finds matches via Exa. Or filter builder (industry, size, location). Or paste CSV. Exa
2
Configure Options Target role (CEO / VP Sales / CMO etc.), auto-push toggle, min quality score threshold. Settings
3
Sequential Processing Per company: find person → fetch signal → generate email + LP → score check → push if above threshold. Live progress table. Exa Groq Apollo
4
Results Table Each row: person found, subject line, P·C·CTA scores, Apollo status. Expand to read full email. LP link per row. Redis LP

POST/GET /api/history Redis: save pushed prospects, read history list
POST /api/push/apollo Create contact + enrol in sequence via Apollo v1
POST /api/push/slack Fire Slack incoming webhook with scores + LP link
POST /api/discover ICP description → Exa company discovery for bulk
POST /api/bulk Orchestrates full per-company run: people → signals → generate → push
GET /lp/[slug] Public prospect landing page — reads content from Redis
External API calls
🔍
Exa
Real-time web search API. Used for: company signal search, person signals, people lookup, ICP discovery, email enrichment.
EXA_API_KEY
⚡
Groq
LLaMA 3.3 70B inference. Generates: email body, 3 subject lines, landing page copy, quality scores (P·C·CTA), angle reasoning.
GROQ_API_KEY
🚀
Apollo.io
CRM v1 REST API. Creates contact record, matches person, optionally enrols in outbound sequence by ID.
APOLLO_API_KEY
💬
Slack
Incoming webhook notification. Fires on push: prospect name, signal used, P·C·CTA scores, landing page link.
SLACK_WEBHOOK_URL
🗄️
Upstash Redis
Serverless key-value store. Stores: prospect history, landing page content by slug, LP → prospect reverse index for visit tracking.
UPSTASH_REDIS_*
Outputs
Generated Outputs
✉️
Cold Email
150–200 word personalised body, referencing the chosen signal. LP URL embedded inline. Editable before push.
📋
Subject Lines
3 variants per generation. Curiosity / Direct / Signal-reference styles. Each with reasoning. User picks one.
🌐
Landing Page
Personalised page at /lp/[slug]. Headline, subheadline, body, CTA. Stored in Redis. Editable. Visit-tracked.
📊
Quality Scores
Personalization · Clarity · CTA — each 1–10. Animated bars. Gate on push if any score below 6.
