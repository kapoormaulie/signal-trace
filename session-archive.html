Project Overview

SignalTrace is a signal-powered cold email tool that finds live buying signals for a B2B prospect, generates a hyper-personalised email and landing page using AI, then pushes the lead to Apollo CRM and Slack — all in one flow.

🔍 Signal Detection

Uses Exa to find real-time news, funding rounds, hiring signals, and personal activity for the prospect.

✉️ AI Email Generation

Groq (LLaMA 3.3 70B) writes a personalised cold email referencing the chosen signal + sender company context.

🚀 CRM Push

One-click push adds the contact to Apollo CRM, enrols in sequence, and sends Slack notification with scores.

🌐 Landing Pages

Each lead gets a personalised landing page (stored in Redis) with dynamic headline, body, and CTA from your sender's perspective.

📊 Bulk Mode

ICP-match, filter builder, or CSV import. Process many companies sequentially with live progress table.

🌙 Dark / Light Mode

Full dark/light theme with CSS variable system. No flash on reload. Preference saved to localStorage.

App Flow

Single Prospect

Entry

Type a company name — Exa finds decision-makers. Or enter prospect manually (name, company, email, LinkedIn).

People picker

Select which person to target from the Exa results grid.

Signal picker

Choose a buying signal (funding, news, hiring, personal activity) or skip to write without one.

Generate

Groq writes a personalised email + landing page + 3 subject line variants + quality scores (personalization, clarity, CTA).

Review & edit

Edit email body and LP content (headline, subheadline, body, CTA text + URL) inline.

Push

Pushes contact to Apollo CRM, enrols in sequence, and fires Slack webhook with scores + LP link.

Bulk Mode

Find companies

ICP description → AI finds matches. Or use filter builder (industry, size, location, funding). Or import CSV.

Configure

Set target role (CEO, VP Sales, etc.), auto-push toggle, minimum quality score threshold.

Generate all

Sequential processing with live progress table. Each row shows person found, subject line, scores, Apollo status.

Key Technical Decisions

DecisionRationale
CSS variables for semantic colorsAllows all Tailwind utilities (text-ink, bg-surface etc.) to flip automatically on data-theme change without touching every component.
ECG as the signature animationThematically perfect (SignalTrace), used sparingly in 3 places — loader, hero, logo — not scattered everywhere. Spends design boldness in one memorable element.
Glass morphism cardsBackground blobs at z-0, glass cards at z-1 let the color bleed through subtly without distraction.
Groq llama-3.3-70b over OpenAI GPTFaster inference, free tier, sufficient quality for email generation.
Override ctaUrl server-sideNever trust LLM to output a correct URL. After parsing, server always sets ctaUrl = sender.defaultCtaUrl. Gate in PushButton if empty.
Upstash Redis for LP storageServerless, no cold-start, REST API works in Next.js edge/serverless functions.
Plus Jakarta Sans (not Inter)Slight techiness that maps to the precision-instrument character of the product. JetBrains Mono only in email textareas.
Sequential bulk processingAvoids rate-limit hammering Exa/Groq/Apollo APIs. User sees live progress per company.

Session 2 — July 1, 2026: Auth, CRM Alternatives, Enrichment, Discovery

Second build session. Focus: giving the CRM-push step real alternatives (not just Apollo), letting the app work across devices/accounts, speeding up bulk, and adding an optional email-enrichment provider. Model: Claude Sonnet 5.

🔗 Alternative CRM (Webhook)

Apollo and a generic CRM webhook (HubSpot/Pipedrive/Salesforce/Zapier/Make) are interchangeable push targets — only one is required. Presented as sub-tabs under one Integrations card, not two separate required cards.

⬇️ CSV / Prospect Reports

Download-a-CSV button on the single push panel, bulk results table, and history page — no integration required, always available.

✉️ Team Email Fallback

If Slack isn't connected, a "Notify team via email instead" link opens a pre-filled mailto: — no backend send, no new API key.

🔐 Optional Auth

Self-built email/password login (scrypt hashing, Redis-backed session cookies, no new services). App still fully works logged-out.

☁️ Account Sync

Logged in: Settings + History follow the account instead of localStorage/device. Logged out: unchanged, per-browser as before.

⚡ Faster Bulk

Bulk generation runs a 4-worker concurrency pool instead of one company at a time — several times faster wall-clock for larger batches.

🔢 Adjustable Discovery Count

"How many companies?" — pills for 10/20/50/100 plus a manual custom number input, clamped 1–100 (was a fixed 20).

🆕 Fresh Set on Re-search

Discovery remembers companies already added to the queue this session and excludes them from the next search (server over-fetches from Exa to compensate).

📧 Hunter.io Enrichment

Optional email finder/verifier, wins priority over Exa/Apollo email resolution when it returns a confidence-scored match. No-ops if HUNTER_API_KEY isn't set.

Key technical decisions

DecisionWhy
Sessions in Redis, not JWTNo new AUTH_SECRET env var, and logout actually revokes access immediately instead of waiting out a token's expiry.
scrypt (Node built-in) over bcryptjsZero new npm dependency — crypto.scrypt + crypto.timingSafeEqual is already in the Node runtime Next.js API routes run on.
Auth is opt-in, not requiredExplicit user decision — SignalTrace stays a fully working local tool if you never log in.
Generic CRM webhook instead of native HubSpot/Pipedrive clientsOne relay works with Zapier/Make/any inbound webhook — broader coverage than building N per-CRM API clients, for far less code.
Only Hunter.io wired in, not BuiltWith/Proxycurl/PeopleDataLabsExplicit user correction — those three don't have a genuine ongoing free tier; user chose to add all four anyway then narrowed to Hunter.io only.
Bulk concurrency fixed at 4, not user-configurableBalances speed against Groq's free-tier RPM limit without adding a new settings control for a "one more knob" that wasn't asked for.
